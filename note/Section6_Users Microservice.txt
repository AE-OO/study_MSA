섹션6. Users Microservice

◆ 전통적인 인증 시스템
1. client에서 서버단으로 인증 요청을 할 때 username과 pwd 같은 데이터를 보냄
2. 서버에서는 db에 있는 데이터인지 검사를 함 => 세션정보를 보냄
3. client는 세션과 쿠키를 이용해서 사용자에 대한 추가적인 작업을 함
4. client에서 보낸 세션정보가 db에서 발급된게 맞는지 확인해서 주문정보 등을 열람할 수 있도록 함

문제 => 세션과 쿠키는 모바일 애플리케이션에서 사용할 수 없음 (공유가 안됨), 모바일에서는 json으로 변환하는 작업이 필요함

◆ token 기반 인증 시스템
1. client에서 서버단으로 인증 요청을 할 때 username과 pwd 같은 데이터를 보냄
2. 서버에서 인증 절차를 거친 다음 token을 발급함
3. client는 token을 가지고 추가적인 작업을 함
